**2.** What is the `Singleton` design pattern?
<!-- enter you answer in the space below -->
```
It is a pattern to ensure that only one instance of a class is created, and provide a global way to access that instance. 
```
**3.** What is the `Observer` design pattern?
<!-- enter you answer in the space below -->
```
The Observer design pattern is a behavioral design that allows an object (the subject) to notify a list of dependent obcects (observers) automatically when its state changes. 
```
**4.** What is the `Strategy` design pattern?
<!-- enter you answer in the space below -->
```
The Strategy design pattern is a way of organizing your code so that you can easily switch between algorithms or behaviors depending on the situation. It works by defining the family of algorithms, each that encapsulates in its own class, and allowing you to switch between them at runtime. 
```
**5.** What is the `Factory` design pattern?
<!-- enter you answer in the space below -->
```
The Factory design pattern is a creational design pattern that lets you create an object without specifying the exact class of the object the created object. It lets you use a factory to create an object for you. 
```
**6.** What is test driven development?
<!-- enter you answer in the space below -->
```
It is a process in which you write automated tests for your code before you actually write code. So before you write any code, you create a test that tells you what the code should do. Then you write the code and see if it can past the test. 
```
**7.** In Scrum/Agile what is the DoD?
<!-- enter you answer in the space below -->
```
DoD stands for Definition of Done. It is a share definition within the scrum team of what done actually means. It ensures that the team is on the same page and that what they deliver each sprint is of high qulaity.
```
